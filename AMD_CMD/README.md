# AMD (RequireJS) vs CMD (SeaJS)

## 目标一致

两种规范都是为了实现模块化协同开发（主要针对浏览器端）。

## 异同对比

```
         | RequireJS                         | SeaJS
------------------------------------------------------------------------------------
书写风格  | 近似于异步回调风格                  | 近似于 CommonJS 风格
------------------------------------------------------------------------------------
依赖加载  | factory 执行前各个依赖必须加载完毕   | 同上
------------------------------------------------------------------------------------
依赖执行  | factory 执行前执行各个依赖,         | factory 执行过程中，用到才执行各个依赖
         | 并作为参数按序传递给 factory 函数。  |
------------------------------------------------------------------------------------
不足之处  | 各依赖模块先于 factory 执行前执行，  | 模块依赖通过 factory.toString() 解析
         | 如果部分模块只有在特定条件下才用到    | 依赖，这就导致 factory 中的 require 模
         | 那么，就造成了不必要的模块执行消耗    | 块名必须硬编码，同时代码压缩必须保证
         | 么，就造成了不必要的模块执行消耗。    | factory 中的 require 参数名不变 
```
